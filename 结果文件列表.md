# 项目代码生成的结果文件列表 (V2 - 完整版)

本文件详细列出了在运行项目代码库（包括重构后的`ComposeProject`和遗留的`Kriging`, `PINN`模块）后，可能会生成的所有结果文件。

---

## 1. 核心图表文件 (可直接用于论文主体)

这些是由代码直接生成的、信息量大、适合直接用于论文的图表。

#### 1.1 PINN训练历史对比图 (来自 `src` 模块)

*   **生成函数**: `ComposeProject/src/analysis/plotting.py` -> `plot_training_comparison()`
*   **触发位置**: 由 `ComposeProject/main.py` 在PINN训练流程结束后调用。
*   **文件名模板**: `results/training_history_{实验名称}.png`
*   **内容描述**: **（最重要）** 以折线图形式展示PINN模型MRE随训练轮数下降的曲线。可以清晰标注出自适应训练过程中的**数据注入**、**Kriging重采样**等关键事件，是验证**第二个创新点（自适应框架）**有效性的核心视觉证据。

#### 1.2 3D重建精度对比图 (来自 `Kriging` 模块)

*   **生成函数**: `Kriging/draw.py` -> `draw_result3D()` -> `draw_unit()`
*   **触发位置**: 需要在测试脚本中手动调用。
*   **文件名模板**: `precision.pdf`
*   **内容描述**: 生成一个包含三个**并排3D散点图**的面板，分别展示**模拟真值 (Simulated)**、**GPU-Kriging预测** 和 **CPU-Kriging预测**。这可以非常直观地对比不同方法的三维重建效果。

#### 1.3 1D轴向切片对比图 (来自 `Kriging` 模块)

*   **生成函数**: `Kriging/draw.py` -> `draw_result()`
*   **触发位置**: 需要在测试脚本中手动调用。
*   **文件名模板**: `x.png`, `y.png`, `z.png`
*   **内容描述**: 沿着X, Y, Z三个主轴的某条线进行切片，绘制**一维折线图**，对比该线上每个点的**预测值**与**真实值**。这对于分析模型在特定方向上的误差分布很有帮助。

## 2. 需少量修改即可生成的关键图表

以下模块中的函数默认使用`plt.show()`进行交互式展示，但只需添加一行`plt.savefig()`即可保存为高质量图表用于论文。

#### 2.1 多平面切片综合对比图 (来自 `PINN` 模块)

*   **生成函数**: `PINN/visualization.py` -> `Visualizer.plot_comparison()`
*   **内容描述**: 生成一个**3x3的组合图**，是信息密度非常高的结果图。它同时展示：
    1.  **MC真值**在XY, XZ, YZ三个平面上的切片。
    2.  **PINN预测**在对应平面上的切片。
    3.  **PINN与MC的相对误差**在对应平面上的分布。
*   **论文价值**: 这是对比模型**空间误差分布**的绝佳图表。

#### 2.2 3D散点采样对比图 (来自 `PINN` 模块)

*   **生成函数**: `PINN/visualization.py` -> `Visualizer.plot_3d_comparison()`
*   **内容描述**: 生成一个**并排的3D散点图**，通过在三维空间中对有效体素进行采样，来对比MC真值和PINN预测的整体三维分布形态。

#### 2.3 变异函数拟合图 (来自 `Kriging` 模块)

*   **生成函数**: `Kriging/myPyKriging3D.py` -> `display_variogram_model()`
*   **内容描述**: 显示Kriging建模的第一步——**半方差理论模型**与**实际数据点**的拟合情况。
*   **论文价值**: 可用于第二章或第三章，作为解释Kriging原理或展示模型配置的插图。

## 3. 数据与模型文件 (过程产物)

#### 3.1 PINN模型检查点

*   **生成逻辑**: `ComposeProject/src/models/pinn.py` 在训练时通过回调函数自动保存。
*   **文件名模板**: `models/pinn_checkpoint-{epoch}.pt`
*   **内容描述**: PyTorch模型文件，用于**断点续训**和**最佳模型恢复**。

#### 3.2 数据集缓存

*   **生成逻辑**: `src/data/loader.py` 或旧版 `dataAnalysis.py` 在首次从Excel读取数据时生成。
*   **文件名模板**: `PINN/DATA_pkl_data/pkl{z}.pkl` 或 `ComposeProject/DATA_pkl_data/pkl{z}.pkl`
*   **内容描述**: `.pkl`文件，用于**缓存Excel数据**，避免重复读取，加速程序启动。

#### 3.3 FLUKA原始数据转换 (预处理)

*   **生成函数**: `Kriging/dataAnalysis.py` -> `extract_matrix_values()`
*   **触发位置**: 需要手动调用。
*   **文件名模板**: (用户自定义).xlsx
*   **内容描述**: 这是一个数据预处理功能，可以将粒子输运软件FLUKA生成的`.lis`文本文件，转换为项目所需的**多Sheet Excel格式 (`DATA.xlsx`)**。
